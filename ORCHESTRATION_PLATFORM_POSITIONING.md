## Category Frame

Foreman occupies a new category: **consumer-grade AI agent orchestration for business.**

We are not a development framework (LangChain), a workflow builder (Zapier/n8n), or a chatbot platform. We are the operating system for a small business AI workforce -- the interface where you hire, direct, supervise, and scale AI agents the same way you'd manage a human team.

**Category name:** AI Agent Orchestration (for business)
**Nearest neighbor:** "What Slack did for communication, Foreman does for AI labor"

## Positioning Statement

**For solopreneurs and small teams who need to scale execution without scaling headcount, Foreman is the agent orchestration platform that gives you a managed AI workforce with a Chief of Staff who coordinates everything. Unlike developer frameworks that require code or point tools that do one thing, Foreman provides an intuitive command-and-control interface where non-technical owners direct, approve, and oversee AI agents the same way they'd manage a human team.**

## Orchestration-Specific Value Pillars

### 1. Chief of Staff Coordination
- **What:** An AI Chief of Staff receives your requests, assigns the right agent, creates execution plans
- **Why It Matters:** You talk to one point of contact, not five agents
- **Differentiator:** No other platform offers a coordinating agent layer

### 2. Plan-Card Approval Flow
- **What:** Before any agent acts, you see the plan and approve it
- **Why It Matters:** AI without oversight is a liability; plan approval keeps you in control
- **Differentiator:** Frameworks run code; Foreman presents choices

### 3. Unified Inbox
- **What:** All agent decisions, completed tasks, and sign-off requests in one place
- **Why It Matters:** You don't check five tools or five agent dashboards
- **Differentiator:** Single pane of glass vs. fragmented tooling

### 4. Org Chart That Grows With You
- **What:** Start with a CEO + Chief of Staff, add Marketing, Sales, Ops agents as needed
- **Why It Matters:** Hire AI agents the same way you'd hire people -- by role, when needed
- **Differentiator:** Role-based org structure vs. workflow nodes
