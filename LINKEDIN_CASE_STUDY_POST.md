1. CTO - Technical planning & implementation

5. Reviewer - Code review & quality

8. Monitoring agent - System health & alerts

Each agent has role-based permissions, monthly budgets, and specialized tool access.

**Real examples from the case study:**
- Shipping dark mode in 2 days instead of 2 weeks
- Automating 80% of marketing content creation
- Reducing support response time from hours to minutes
- Cutting operational coordination meetings by 80%

**We made mistakes too (and fixed them):**
- The $12 optimization bug (fixed in 48h)
- Pricing typo in our first public-facing page
- Over-provisioning 47 integrations (now capped at 10)
- Underestimating token costs for certain workflows

**Technical architecture highlights:**
- Backend API on Railway (scales to 500+ concurrent)
- Supabase for data layer with real-time sync
- Paperclip for agent orchestration & governance
- OpenClaw for local agent execution
- Role-based model routing (DeepSeek, Qwen3, etc.)
- Monthly budget caps per agent (no overspending)
