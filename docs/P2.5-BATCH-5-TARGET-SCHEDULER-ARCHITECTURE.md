# P2.5 Batch 5 - Target Scheduler Architecture (Paperclip Era)

Date: 2026-04-08
Batch task: B5.2

## Objective

Replace fragmented launchd/cron/mixed-loop scheduling with a single Paperclip-oriented scheduling contract for Foreman v2 automation.

## Target architecture

1. **Single scheduler authority**
   - Paperclip-runner scripts in `foreman-v2/scripts/` are the only supported batch entrypoints.
   - Legacy `foreman/` launchd/cron loops remain reference-only during migration.

2. **Explicit job catalog**
   - Each recurring automation job has:
     - `job_id`
     - owner agent
     - cadence
     - entry command
     - success criteria
     - failure policy

3. **Evidence-first execution**
   - Every scheduled run writes a JSON artifact under `foreman-v2/state/` with:
     - timestamp
     - target agent/job
     - run id
     - status
     - stderr/stdout excerpt

4. **No silent retries**
   - Retry policy must be explicit per job.
   - Terminal failures must remain visible in artifact output.

5. **Role-routing compatibility**
   - Scheduled jobs invoke Paperclip agents that already use `paperclip-role-dispatch.sh`.
   - Role routing remains sourced from `config/role-routing.json`.

## Initial migration shape

- Pilot one legacy-style recurring function into a Paperclip scheduler entrypoint.
- Validate parity with a reproducible harness.
- Split remaining automation backlog into follow-up slices (P2.5a/P2.5b) after pilot.
