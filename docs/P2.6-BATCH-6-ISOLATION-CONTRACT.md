# P2.6 Batch 6 - Isolation Requirements and Threat Model

Date: 2026-04-08
Scope tasks: B6.1, B6.2, B6.3

## B6.1 Isolation requirements and threat model

### Required tenant boundary

- A tenant is represented by a Paperclip company context.
- No task execution may cross tenant/company boundary unless explicitly authorized by a human escalation path.
- Tenant-scoped execution artifacts must remain attributable to one tenant.

### Threat model (minimum set)

1. **Cross-tenant execution**: command for tenant A executes under tenant B context.
2. **Cross-tenant data read/write**: state artifact or run output from tenant A appears in tenant B context.
3. **Stale context reuse**: previous tenant context leaks into a new run due to reused process/session state.
4. **Misbound credentials**: tenant B credentials are applied to tenant A run path.
5. **Silent fallback boundary loss**: failure path defaults to shared/global context without explicit error.

### Mandatory controls

- Every execution path must carry explicit tenant identity from trigger to runtime.
- Context mismatch must fail loud (non-zero/error status), not auto-correct silently.
- Validation harness must include at least one negative isolation test.

## B6.2 Company context propagation contract

Tenant identity contract fields:

- `paperclip_company_id` (primary tenant key)
- `paperclip_agent_id` (agent identity within tenant)
- `paperclip_run_id` (execution instance key)
- `tenant_execution_key = paperclip_company_id + ":" + paperclip_run_id`

Propagation rules:

1. Trigger layer provides `paperclip_company_id` and `paperclip_agent_id`.
2. Worker/dispatch layer receives tenant fields as env/context and includes them in session identifiers.
3. Evidence artifact includes tenant fields for every recorded run.
4. If `paperclip_company_id` is missing, execution aborts with explicit error.

## B6.3 Config/state separation plan

### Shared (global) config

- Routing contract: `config/role-routing.json`
- Static docs/runbooks under `docs/`
- Reusable scripts under `scripts/`

### Tenant-scoped state (must remain isolated)

- Per-run evidence JSON for tenant execution (store tenant id in payload)
- Paperclip company-specific runs/issues/agents
- Any future tenant-specific policy overrides

### Migration notes

- Existing state artifacts should be extended with tenant identifiers where absent.
- New scheduler/isolation scripts must write tenant metadata by default.
