# P2.3 Batch 3 - Role Boundaries (First Slice)

Date: 2026-04-08
Batch task: B3.3
Scope: first hierarchy slice (`ChiefOfStaff`, `EngineeringBuilder`, `DevOpsAgent`, `QAEngineer`)

## Responsibility boundaries

| Role | Owns | Must not own |
|---|---|---|
| `ChiefOfStaff` | Coordination, prioritization, delegation across the slice; escalation to CEO | Direct implementation work, infra operations, test execution |
| `EngineeringBuilder` | Feature and code implementation tasks delegated from ChiefOfStaff | Project-wide coordination, final QA sign-off, infra runbook ownership |
| `DevOpsAgent` | Runtime and infrastructure operations delegated within scope | Product-spec ownership, broad code implementation, QA acceptance authority |
| `QAEngineer` | Test execution, verification evidence, failure triage reports | Production infra changes, feature implementation, staffing/coordination decisions |

## Delegation contract

- Delegation flows from `ChiefOfStaff` to specialist roles in this slice.
- Specialists report completion evidence back through the issue hierarchy (parent/child issues).
- Cross-domain requests are routed back to `ChiefOfStaff` for reassignment, not handled ad hoc.

## Non-overlap rules

- No specialist directly reassigns another specialist without ChiefOfStaff visibility.
- QA does not patch production behavior directly; QA reports and verifies.
- DevOps does not redefine product requirements; infra changes stay requirement-driven.
- EngineeringBuilder does not bypass QA verification for completion claims.

## B3.3 completion output

- Role boundaries defined and documented for the first migration slice.
- Delegation contract and non-overlap rules captured for downstream hierarchy expansion.
